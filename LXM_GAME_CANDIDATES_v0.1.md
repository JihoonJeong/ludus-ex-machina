# LxM Game Candidates — Evaluation & Priority

**Date:** 2026-03-12
**Purpose:** 다음에 추가할 게임 후보를 객관적으로 평가하고 우선순위를 정한다.
**현재 게임:** Tic-tac-toe, Chess

## 1. 평가 기준 (6축, 각 5점 만점)

| 축 | 약어 | 설명 | 점수 기준 |
|----|------|------|----------|
| **구현 가능성** | IMPL | 기존 라이브러리, 규칙 명확성, validation 결정론성 | 5=하루, 4=며칠, 3=1-2주, 2=수주, 1=대규모 |
| **AI 차별화** | DIFF | 모델 간 실력/행동 차이가 드러나는 정도 | 5=극적 차이, 1=거의 동일 |
| **관전 재미** | VIEW | 뷰어에서 보는 재미, 컨텐츠 가치 | 5=유튜브급, 1=로그나 다름없음 |
| **카테고리 신규성** | NEW | 기존 LxM 게임과 다른 유형을 여는가 | 5=완전 새 카테고리, 1=기존과 동일 |
| **문화/상업 매력** | APPEAL | 인지도, 글로벌/한국 시장 매력 | 5=누구나 안다, 1=니치 |
| **MM 데이터 가치** | MM | Model Medicine 연구에 새로운 측정 축을 제공하는가 | 5=새 MTI 축 후보, 1=기존 데이터 반복 |
| **Shell 경쟁 가치** | SHELL | 사용자가 전략 문서(Soft Shell)를 만들어서 주입했을 때 결과가 의미있게 바뀌는가 | 5=공략집이 승패를 좌우, 1=Shell 무효 |

**총점 = 단순 합산 (35점 만점)**

단, 카테고리 신규성(NEW)에 **1.5x 가중치**를 적용. 이유: 같은 카테고리 게임을 여러 개 추가하는 것보다 다양한 유형을 확보하는 게 플랫폼에 더 가치있음.

**가중 총점 = IMPL + DIFF + VIEW + (NEW × 1.5) + APPEAL + MM + SHELL (최대 37.5점)**

**SHELL 축 추가 이유 (v0.1 → v0.2 변경):** LxM은 연구 플랫폼이자 제품. "AI coaching esports" — 사용자가 전략 문서를 설계해서 에이전트에 주입하고 경쟁하는 것이 핵심 사용자 경험. SIBO Spectrum 데이터에서 확인된 것처럼, Shell 영향력은 게임마다 극적으로 다름 (Trust Game 0.75, Codenames 0.35, Chess 0.10). Shell 경쟁이 성립하지 않는 게임은 제품 가치가 낮음.

**SHELL 점수 기준:**
- 5 = Shell이 전략/행동을 근본적으로 바꿈. 공략집이 승패를 좌우. (예: Avalon — 스파이 탐지 전략)
- 4 = Shell이 스타일을 뚜렷하게 바꿈. 다양한 공략이 가능. (예: Poker — 블러핑 전략)
- 3 = Shell이 부분적 영향. 일부 전략만 효과. (예: Codenames — 클루 스타일 변화)
- 2 = Shell이 미미한 영향. 대부분 Core가 지배. (예: Chess)
- 1 = Shell이 거의 무효. Core 전문성이 완전 지배.

## 2. 후보 게임 평가

- **규칙:** 15×15 보드, 2인, 교대로 돌 놓기, 5연속 승리
- **라이브러리:** 없지만 로직이 tic-tac-toe 확장
- **평가:**

| IMPL | DIFF | VIEW | NEW | APPEAL | MM | 가중 총점 |
|------|------|------|-----|--------|-----|----------|
| 5 | 3 | 2 | 1 | 3 | 2 | 16.5 |

- **장점:** 반나절 구현. 게임 수 빠르게 확장. 뷰어 렌더러도 거의 공짜.
- **단점:** Chess와 같은 카테고리. AI 차별화도 Chess보다 약할 수 있음. 관전 재미 낮음 — 돌 놓는 게 체스 기물 이동보다 시각적으로 단조로움.
- **MM 가치:** 공간 패턴 인식 측정은 가능하지만 Chess 데이터와 크게 다르지 않음.

#### A2. 바둑 (Go) 9×9
- **규칙:** 9×9 보드 (표준은 19×19), 교대로 돌 놓기, 영토 확보
- **라이브러리:** python 라이브러리 여럿 (gnugo 바인딩, sgfmill 등). 집 계산이 까다로움.
- **평가:**

| IMPL | DIFF | VIEW | NEW | APPEAL | MM | 가중 총점 |
|------|------|------|-----|--------|-----|----------|
| 3 | 5 | 4 | 1 | 5 | 4 | 22.5 |

- **장점:** AI 역사의 상징 (AlphaGo). 전략 깊이가 체스보다 깊어서 모델 간 차이 극적. 글로벌 인지도 최고.
- **단점:** 같은 카테고리 (완전정보 턴제). Scoring(집 계산)이 복잡 — 중국식/일본식 규칙 차이. 9×9로 시작하더라도 구현 쉽지 않음.
- **MM 가치:** 장기적 전략 계획 능력 측정. 체스보다 "직관적 판단" 의존도가 높아서 다른 데이터.

#### A3. 장기 (Janggi)

- **라이브러리:** python-janggi (존재 여부 확인 필요). 직접 구현 가능.

- **MM 가치:** "같은 모델이 서양 체스 vs 동양 장기에서 다르게 행동하는가" — 문화적 학습 편향 측정. 있지만 niché.

### Category B: 불완전정보 카드게임 (새 카테고리)

#### B1. 포커 (Texas Hold'em)
- **규칙:** 2-6인, 홀 카드 2장 + 공용 5장, 베팅 라운드, 족보 비교
- **라이브러리:** pokerkit, treys (핸드 평가), deuces 등 풍부
- **평가:**

| IMPL | DIFF | VIEW | NEW | APPEAL | MM | 가중 총점 |
|------|------|------|-----|--------|-----|----------|
| 4 | 5 | 5 | 5 | 5 | 5 | 31.5 |

- **장점:** 모든 축에서 높은 점수. 불완전 정보 + 블러핑 + 확률 계산 + 상대 모델링 — 완전히 새로운 카테고리. 글로벌 인지도 최고. 학술적으로도 핫 (Pluribus, Libratus). 관전 재미 검증됨 (WSOP, 스트리밍).
- **단점:** 멀티플레이어(2+)를 지원해야 함. 베팅 시스템 구현이 체스보다 복잡 (팟, 사이드팟, 올인 규칙 등). 운의 요소가 커서 적은 판 수로는 실력 비교 어려움 — 통계적으로 유의미하려면 많은 판 필요.
- **MM 가치:** 블러핑 능력 (Sociality의 새 하위축), 확률 추론 (Reactivity), 손실 후 행동 변화 (Resilience), 상대 패턴 학습 (ICL 능력). MTI의 거의 모든 축에 데이터 제공.

## 4. 동적 평가와 최종 구현 순서

### 정적 평가의 한계

Section 3의 점수는 **정적 평가** — 모든 게임을 독립적으로 채점함. 하지만 NEW(카테고리 신규성) 점수는 **동적**이어야 함. Coup(불완전정보+블러핑)을 먼저 구현하면 포커의 NEW가 5→2로 떨어지고, 포커를 먼저 구현하면 Coup의 NEW가 5→2로 떨어짐. 같은 카테고리 안에서 두 번째 게임은 "카테고리 개척" 보너스를 받을 수 없음.

### 원칙: 카테고리 대표 게임 우선

따라서 순서 결정의 핵심 원칙: **각 카테고리에서 가장 강한 대표 게임을 먼저 구현하고, 같은 카테고리 두 번째 게임은 나중에 "깊이 추가"로.**

| 카테고리 | 같은 카테고리 후보들 | 대표 게임 | 이유 |
|---------|-------------------|----------|------|
| 완전정보 턴제 | Chess ✅, 오목, 바둑, 장기, 오셀로, Hex | 이미 완료 | — |
| 불완전정보+블러핑 | **포커**, Coup, 블랙잭 | **포커** | 인지도, 깊이, 관전 전부 최고. Coup은 포커의 단순화 버전 |
| 팀 협력+언어 | **Codenames** | **Codenames** | 유일한 후보. LLM 본질 테스트 |
| 게임이론 | **Trust Game** | **Trust Game** | 유일한 후보. 구현 trivial |
| 사회적 추론 | **Avalon**, 마피아 | **Avalon** | 마피아보다 구조화, validation 가능 |
| 경제+협상 | **모노폴리**, 경매, Negotiation Arena | **모노폴리** | 인지도 압도적, 컨텐츠 가치 최고 |
| TRPG | **D&D 크롤** | **D&D 크롤** | 유일한 후보. 다축 MM 데이터 최고 |
| 협력 | **Hanabi** | **Hanabi** | 유일한 협력 게임. 새 MM 축 |
| 실시간 | **Snake** | **Snake** | tick_based 모드 검증용 |

같은 카테고리 두 번째 게임(Coup, 마피아, 경매 등)은 해당 카테고리의 대표 게임이 안착한 후 수요에 따라 추가.

### 동적 점수 기반 구현 순서

매 단계에서 "아직 안 열린 카테고리"의 대표 게임 중 가중 총점이 가장 높은 것을 선택:

| 순서 | 게임 | 정적 점수 | 여는 카테고리 | 구현 난이도 |
|------|------|----------|-------------|------------|
| **1** | **포커 (Texas Hold'em)** | 31.5 | 불완전정보+블러핑+확률 | 1-2주 |
| **2** | **Codenames** | 28.5 | 팀 협력+언어 | 1주 |
| **3** | **Trust Game** | 25.5 | 게임이론/협력-배신 | 1일 |
| **4** | **Avalon** | 28.5 | 사회적 추론+속임수 | 1-2주 |
| **5** | **모노폴리** | 27.5 | 경제+협상+운 | 2-3주 |
| **6** | **D&D 던전 크롤** | 28.5 | TRPG/어드벤처 | 3-4주 |
| **7** | **Hanabi** | 25.5 | 협력 게임 | 1주 |
| **8** | **Snake (멀티)** | 24.5 | 실시간 (tick_based) | 1주 |

Trust Game이 3번인 이유: Codenames(1주) 구현 중 병렬로 하루 만에 완성 가능. 게임 수를 빠르게 늘리면서 새 카테고리도 열음.

### 카테고리 확장 후 깊이 추가 (필요시)

대표 게임이 안착한 카테고리에 두 번째 게임을 추가해서 깊이를 더하는 단계. 우선순위는 사용자 수요와 연구 필요에 따라:

| 카테고리 | 추가 후보 | 조건 |
|---------|----------|------|
| 불완전정보 | Coup, 고스톱 | 포커 안착 후. Coup은 빠르게, 고스톱은 한국 시장 확장 시 |
| 완전정보 | 바둑, 장기, 오셀로 | 수요 있을 때. 바둑은 상징성으로 우선 |
| 사회적 추론 | 마피아 | Avalon 안착 후. 더 자유로운 형태의 데이터 필요 시 |
| 경제 | 경매, Negotiation Arena | 모노폴리 안착 후 |
| TRPG | 소셜 어드벤처 | D&D 크롤 안착 후 |
