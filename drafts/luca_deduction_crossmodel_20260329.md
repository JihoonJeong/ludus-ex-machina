## 핵심 발견

### 1. Opus 3/3 — 추리는 모델 크기가 중요
유일하게 모든 난이도 통과. 더 많이 읽고(5-8파일), 더 정확하게 추론. 포커/아발론에서는 tier가 결과를 안 바꿨는데, **추리에서는 모델 크기가 결정적.**

### 2. 난이도 체계가 모델마다 다르게 작동
- Easy(001): Haiku/Sonnet 실패, Opus만 통과 → "Easy"가 실제로 쉽지 않음
- Medium(002): Haiku 통과, Sonnet 실패 → 비직관적
- Hard(003): Sonnet 통과, Haiku 실패 → 예상대로

**난이도가 모델 능력과 비선형적으로 상호작용.** 특정 시나리오의 단서 구조가 특정 모델의 추론 패턴과 맞을 수 있음.

### 3. Sonnet의 불일치
- 이전 자유 서술 테스트: 002에서 범인 맞춤
- 이번 선택지 테스트: 002에서 0점 (범인 틀림)
- 003에서는 0파일 읽고 만점 (4.50)

**같은 모델이 같은 시나리오에서 일관되지 않음.** 이건 1회 테스트의 분산 문제 — 통계적 신뢰를 위해 복수 실행 필요.

### 4. 탐색 전략 차이
- Opus: 많이 읽고 확실하게 (5-8파일) → 안정적으로 정답
- Sonnet: 적게 읽고 직관적 (0-3파일) → 맞으면 최고 효율, 틀리면 0점
- Haiku: 중간 (2-5파일) → 불안정

### 5. 포커/체스 vs 추리 — 능력 축이 다름
| 게임 | 모델 크기 영향 |
|------|--------------|
| 체스 | 없음 (Gemini가 tier 무관 압도) |
| 포커 | 없음 (Haiku > Sonnet > Opus) |
| 코드네임 | 없음 (35% 동일) |
| **추리** | **있음 (Opus >> Sonnet = Haiku)** |

**추리가 LxM의 첫 번째 "모델 크기가 중요한" 게임.** 이건 논문에서 중요한 발견 — 능력 축에 따라 모델 크기 효과가 달라짐.
